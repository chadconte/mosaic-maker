5. FAMILY-AWARE COLOR MATCHING
This is a major quality improvement.

During initial color matching, do NOT let every color match to every other color blindly.

Use a family compatibility map like this:

neutral -> neutral
skin -> skin, brown
brown -> brown, skin, red
red -> red, pink, orange, brown
pink -> pink, red, purple
orange -> orange, red, yellow
yellow -> yellow, orange
green -> green
blue -> blue, cyan, purple
cyan -> cyan, blue, green
purple -> purple, pink, blue
metallic -> metallic

Implementation guidance:
- First determine the nearest palette color globally by LAB distance
- Then constrain or bias matches using the family compatibility rules
- A practical approach is:
  a. find the nearest color
  b. get its family
  c. only compare against colors in compatible families
- Use this logic in BOTH:
  1. initial image pixel mapping
  2. threshold cleanup remapping

This is to prevent bad substitutions like reds collapsing into browns too aggressively.

6. CONTRAST-BASED EDGE PROTECTION
Add an option:
- protectEdges: boolean
Default = true

During threshold cleanup:
- compute a simple contrast mask from the already resized image
- if a pixel differs strongly from its neighbors, mark it as protected
- protected pixels should not be aggressively remapped during threshold cleanup

Simple implementation is fine:
- compare pixel RGB or LAB to left/right/up/down neighbors
- if contrast exceeds a threshold, mark protected

Use edge protection only during cleanup, not during the initial mapping.

Goal:
- preserve face details
- preserve text edges
- preserve outlines and highlights
